Analysis Phase¶

1.1 Preliminary Problem Definition¶

This phase is the first approach to the problem, to have a general understanding and a first overlook

The goal is to predict the tracks that would be listened, given the features in the dataset. The dataset is cross sectional. The evaluation metric is a ROC-AUC curve.

So the framework would be that of a binary classification model to approximate the music preference function for a user.

Checking for the structural integrity of the dataset, the statistical consistency of our feature and to look at the context they are used. The goal of this phase is analyze the statistical properties of the dataset and to familiarize with the features, and group them in a contextual perspective.

def NaN_counter (df):
    """
    DataFrame parser for NaN:
    input: df = dataframe
    prints the number of NaN by columns
    output: returns the same df with the number of NaNs per column
    """
    print ("Running NaN Counter from the util.py module!!!!")
    import seaborn as sns
    import pandas as pd

    col = df.columns
    cont = []
    print("NaN Counter by columns")
    for i in col:
        n = len(df[(df[i].isnull() == True)])
        cont.append(n)
        if ( n != 0):
            print(i,":",len(df[(df[i].isnull() == True)]))
    df_r = pd.DataFrame([cont], columns=col)

    try:
        col_d = df_r.columns
        sns.barplot(data=df_r, orient='h').set_title('NaN per feature')
        plt.xticks(rotation=90)
    except:
        pass
    return df_r

1.2.3 Feature Analysis¶

The dataset is clean and intact but there are several problems to be solved before the modeling phase.

The first one is the data imbalance for the single observation:

From a context perspective the dataset is comprised of user-oriented features and song-oriented features, plus the dependent variable. It is important to notice that it represents several users across several songs, but since the model would be approximating the preference's function for a single user, we can assume that those are somewhat constant for different songs. One of the most alarming discrepancies is the difference between the min-max of entries per user. -Given time I would understand the preferences, profile the users and cluster in groups omogenous groups.-

A solution will be downsampling users with extremely high entries to avoid building a model that overepresents the preferences of a single one. To go about it I will restrict the dataset only to users around a certain range of entries ( plus or minus one standard deviation from the mean)

The second problem are the categorical non-ordinal variables. To run meaningful descriptive statistics I'd need to separate the numerical non-ordinal variables, from the numerical ordinal ( duration, ts_listen, age).

Ultimately to avoid the misrepresentation trap, I'd encode the numerical non-ordinal.

To understand the informative value carried by the single non-ordinal features, I'd look at the numerosity of levels per each variable. It turns out that there's a marked discrepancy among them:

1.3 Research Question and Theory¶

1.3.1 Model Selection¶

The literature about the "music preferences", while providing some insight on feature's importance (Schfer 2009), doesn't define an ideal preference function. Its variability though it's very high in some of the studies and strongly depends on the genre and the usage ( Lee 2004, Lamont 2010). According to that we could relax our assumption of a constant preference function per user and keep the user-id imbalance in the dataset.

I will opt for Decision Tree model ( Random Forest - XG Boost) as a benchmark, as it grants more control over the feature's importance. A Neural Network (14 input layer to a 2 neuron output) would be a second choice, altough the dataset is way too small.

References:

Schfer, T. and Sedlmeier, P., 2009. From the functions of music to music preference. Psychology of Music, 37(3), pp.279-300.

Lee, J.H. and Downie, J.S., 2004, October. Survey Of Music Information Needs, Uses, And Seeking Behaviours: Preliminary Findings. In ISMIR (Vol. 2004, p. 5th).

Lamont, A. and Webb, R., 2010. Short-and long-term musical preferences: what makes a favourite piece of music?. Psychology of Music, 38(2), pp.222-241.

1.3.2 Categorical Encodings¶

For how to deal with categorical encodings we will refer to Potdar (2017).

Decision Trees' result are consistent with categoricals and ordinal variables. Altough an encoding is needed to convert the numerical non-ordinal into categorical non-ordinal. I'll try several methods of encodings The first one will be this "simple" conversion to change integers into labels. It will be used as a benchmark.

The second one will be binary encodings as recommended by Potdar (2017). It is a better choice than the one-hot encodings for Decision Tree models because, even though it doesn't solve the problem of the discrepancies between levels across the variables, it is less dense and avoids the curse of dimensionality problem.

References:

Potdar, K., Pardawala, T.S. and Pai, C.D., 2017. A comparative study of categorical variable encoding techniques for neural network classifiers. International Journal of Computer Applications, 175(4), pp.7-9.

1.3.3 Logistic Encodings¶

As a third encoding option, I'd like to try and convert the numerical non-ordinal as numerical ordinal by substituting every label for each feature with its percentage likelihood of being closer to 1 or 0.
